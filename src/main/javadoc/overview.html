Sudoku documentation

Short summary for the Sudoku game

Structure

The structure of the game is separated to 3 parts

  1. sudoku

    Contains the logic of the game and can be divided to parts

    1. SudokuGenerator

      Purpose of this class is to handle building of sudoku

    2. SolveHinter

      This class is to provide artificial solving/hinting without player

    3. Validator

      And last most important is validator which handles validating if sudoku is still solvable

    In Game class all these 3 are connected. Game provides model of game with logic from start to end

  2. IO

    Handles the input and output

    1. Writer

      Writes on terminal, helps to show the board

    2. Reader

      Reads from terminal and parse the message

    3. Option

      Enum for tokenizing the readings

  3. App is the main part of the project, in this class are connected logic of the game with the input/output classes
